1. Executive Summary

This audit presents a thorough evaluation of the live digital presence of the **InAmigos Foundation** (`https://inamigosfoundation.org.in`). Operating as an active Section 8 NGO, the foundation's landing site is its primary brand tool for securing CSR sponsorships, organizing local volunteers, and accepting public donations.

The core focus of this analysis is dual-faceted: identifying usability friction points that degrade brand confidence and diagnosing serious structural security concerns. In addition to assessing layout alignments, counter label typography, and mobile responsive grid scaling, we address a critical template hijacking issue currently active on the live site.

Key Usability & Security Objectives:

The goal is to move the organization's landing site away from generic, cluttered layout systems and vulnerable templates, transitioning toward a premium, modern, glassmorphic UI. This report highlights how these changes are fully realized in the proposed **Task 1 Code Redesign**.

2. Critical Security Analysis

Threat Status: Critical Malware Link Injection Active

Our code inspection revealed active, malicious link injections within the live web templates. Hidden container tags (`<div>` blocks styled absolute off-screen) have been embedded, loaded with keywords targeting online gambling and pharmaceutical sites.

Below is an exact snippet extracted from the live site's DOM source code: 

<!-- Discovered SEO Spam Injection Block -->
<div style="position: absolute; left: -9999px; top: -9999px;">
  <a href="https://shopnoit.com/">online gambling slot</a>
  <a href="https://webwideit.solutions/">trusted online casino bd</a>
  <a href="https://www.perfumelabsbd.com/">slot game online</a>
</div>

Negative Impacts on NGO Credibility

  • Search Engine Index Penalties: Crawlers detect these spam links and immediately flag the site's authority. This severely hurts organic rankings, making search queries for the NGO's projects fail.
  • Browser Safety Blocklists: If left unaddressed, modern browsers like Google Chrome and Firefox will flag the domain as compromised, throwing a red warning screen ("Deceptive site ahead") to donors.

Mitigation & Clean-up Roadmap

Mitigation requires scanning global layouts (like WordPress header files, template engines, or theme directories), stripping out all off-screen hidden divs, updating database tables, resetting SSH/FTP credentials, and deploying a secure web application firewall (WAF).
